# White Room RFP Document

## 1. AG-UI Architecture

This section will detail the overall architecture of the AG-UI, including its key components, their interactions, and the technologies used. It will cover aspects such as frontend framework (React/Vite), state management (Zustand), data fetching (TanStack Query), and styling (TailwindCSS). Special attention will be given to the integration with the backend services and the audio engine.

## 2. Privacy Redaction Rules

This section will outline the privacy redaction rules implemented within the AG-UI and its backend. It will describe how sensitive user data, tool arguments, and other potentially private information are handled, redacted, and anonymized to ensure user privacy and compliance with data protection regulations. Details on server-side redaction enforcement and audit logging will be included.

## 3. Environment Variables

This section will list and describe all environment variables required for configuring the AG-UI and its associated backend services. It will cover variables for API keys, database connections, authentication settings (Clerk), feature flags, and other configurable parameters. Default values and production considerations will be discussed.

## 4. Developer Setup Instructions for CopilotKit Integration

This section will provide comprehensive instructions for developers to set up their local environment for working with the AG-UI, specifically focusing on the integration with CopilotKit. It will include steps for installing dependencies, configuring API keys, running local development servers, and testing the CopilotKit functionalities. Best practices for extending and debugging CopilotKit integrations will also be covered.
